Check Out Procedure

  1. Click on Status for available servers.
  2. Select your username under Assigned To
  3. Change the Status with the following results.
    1. Checked Out: sets the Due Date to TODAY + 1 Week. (Default)
    2. Quick Test: sets the Due Date to TODAY + 1 Day.
    3. Short Test: sets the Due Date to TODAY + 2 Weeks.
    4. Long Test: sets the Due Date to TODAY + 4 Weeks.
    5. Static: erases the Due Date.
    6. Call: erases the Due Date, must use Edit to change Owner or Status.
    7. Appliance|Console: must use Edit to change Owner, Status, or Due Date.
    8. All other Status changes do not affect the Due Date.
    9. NOTE: You can always click Edit to override any of these settings.
  4. Select the Current Test OS you will be using for your test.
  5. When you are finished, please clear your username.
    1. An email will be sent out on Mon and Thu for overdue resources.
    2. NOTE: If the resouce is used in a static configuration, please reboot back into the static configuration when you are done.
  6. If someone has the machine you need, call them before you use it.
  7. WARNING: Do not install over the top of a partition that is being used in a static configuration. Click Yes in the Used in Static Config field to see which partitions are used in the static configuration.
  8. Normally you will only use the active resourses listed. If you want to see all resources and their details, click on List All Resources.
  9. If you are not in the user list, please call the mentor line (4-4977) to check out a server.
